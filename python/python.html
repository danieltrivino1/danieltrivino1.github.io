  • Paths

    Absolute Path:

    An absolute path refers to the complete details needed to locate a file or folder, starting from the root element and ending with the other subdirectories.

    'C:/Users/John/Documents/file1.pdf'

    Relative Path:

    A relative path refers to the location that is relative to a current directory.

    1. Current Directory: if your Python script is located in C:/Users/John/Documents/, and you have a PDF named file1.pdf in the same directory, you can use:
      './file1.pdf' or 'file1.pdf'
    1. Subdirectory: if your Python script is located in C:/Users/John/Documents/, and you have a PDF named file1.pdf in C:/Users/John/Documents/extra_files, you can use:
      './extra_files/file1.pdf'
    1. Parent Directory: if your Python script is located in C:/Users/John/Documents/, and you have a PDF named file1.pdf in C:/Users/John/extra_files, you can use:
      '../extra_files/file1.pdf'

    Note: The way you specify paths might slightly differ based on the operating system. The examples above use forward slashes (/), which work on both Windows and Unix-based systems. However, traditionally, Windows uses backslashes (\). If you use backslashes in Python strings, remember to escape them (\\) or use raw strings (r'C:\path\to\directory').

  • Python¬†Threading

    CPU Bound vs I/O Bound:

    The terms "CPU bound" and "I/O bound" describe the limiting factors of a process or a program based on its resource usage and interaction patterns:

    1. CPU Bound: it is a process that is limited by the CPU speed as it spends most of its time performing computational tasks. Multiprocessing would help.
    1. I/O Bound: it is a process that is limited by I/O (Input/Output) operations. This means the process spends more time waiting for I/O operations to complete than to execute its own computation. Threading would help.

    A thread is a separate flow of execution. By using multiple threads we can speed up applications which face an input/output based bottleneck, a good example of this would be a web crawler.

    Note: the threads are not run independently at the same time (this is not parallel computing). However, with threads you don‚Äôt have to wait for one function to finish to run another. (While threads may appear to run simultaneously, only one thread can be executed at a time. This is enforced by Python‚Äôs¬†global interpreter lock).

        print(f"Main Thread: {j}")

def loop_2(thread_number, num_iterations):

        print(f"{thread_number} Thread: {i}")
        time.sleep(1)

# Number of iterations for loop_2
num_iterations = 5

with concurrent.futures.ThreadPoolExecutor(max_workers=2) as executor:
    # Submit two tasks to the executor with different thread_numbers but the same num_1
    futures = [executor.submit(loop_2, thread_number, num_iterations) for thread_number in range(2)]

    # Wait for all futures to complete
    for future in concurrent.futures.as_completed(futures):
        future.result()  # You can handle results or exceptions here

  • Lambda

    Lambdas are used to create simple functions in an efficient way.

    Note: a lambda function can take any number of parameters, but can only have one expression. They are called anonymous functions in other languages because we don‚Äôt give them names.

    Lambda Syntax:

    Function_Name (Result) = lambda parameters : expression

    Example:

    x = lambda a: a + 10  # Same as: (lambda a: a + 10)(5)
print(x(5))

# Result is 15

y = lambda a,b: a * b  # Same as: (lambda a,b: a * b)(3,4)
print(y(3,4))

# Result is 12

    Lambda (list + map):

    The¬†map()¬†function runs a lambda function over a list. To work with map(), the lambda should have one parameter representing one element from the source list. Choose a suitable name for the parameter, like¬†n¬†for a list of numbers,¬†s¬†for a list of strings. The result of map() is an "iterable" map object which mostly works like a list, but it does not print. Therefore, the examples wrap the map() result in list() to print:

    >>> map(lambda n: n * 2, [1, 2, 3, 4, 5])
<map object at 0x10ce142e8>

>>> list(map(lambda n: n * 2, [1, 2, 3, 4, 5]))
[2, 4, 6, 8, 10]

>>> list(map(lambda x,y: x+y, [4,5,6],[1,3,4])))
[5, 8, 10]

  • Signature of a Method

    Signature:

    Signature refers to a method definition, including its name along with the number and types of its parameters. Signature is crucial because it determines how a method is identified and distinguished from others in the same class (overloading) or in a subclass (overriding).

    Method signature in Python:

    1. Method Name: The identifier that is used to call the method.
    1. Parameters:
      • Number of parameters: How many arguments the method accepts.
      • Types of parameters: While Python is dynamically typed and does not enforce type annotations, you can optionally specify expected data types using Python's type hints.
      • Parameter names: The variable names that will receive the method's arguments within its body.

    Parameters:

    In Python function definitions, positional parameters should be defined before keyword parameters. This rule is necessary because positional arguments are matched based on their order, whereas keyword (or named) parameters are matched by their name.

    Correct order when defining a function:

    1. Standard positional parameters (pos1 and pos2).
    1. Default positional parameters (pos3).
    1. *args to capture additional positional parameters.
    1. Keyword (named) parameters (kw1).
    1. Default keyword parameters (kw2).
    1. **kwargs to capture additional keyword parameters.
    def example_func(pos1, pos2, pos3='default', *args, 
						     kw1, kw2='kw_default', **kwargs):
    print(f"pos1 = {pos1}")
    print(f"pos2 = {pos2}")
    print(f"pos3 = {pos3}")
    print(f"args = {args}")
    print(f"kw1 = {kw1}")
    print(f"kw2 = {kw2}")
    print(f"kwargs = {kwargs}")

    Using *args:

    It allows a function to accept any number of positional arguments beyond those explicitly defined. It collects them into a tuple.

    def function_with_args(*args):
    for arg in args:
        print(arg)

function_with_args('Hello', 'world', 'from', 'Python')

    Using **kargs:

    It allows a function to accept any number of keyword arguments beyond those explicitly defined. It collects them into a dictionary.

    def function_with_kwargs(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

function_with_kwargs(first='Hello', second='world', third='from', fourth='Python')
  • Overriding and Overloading

    Overriding:

    Overriding refers to the ability of a subclass to provide a specific implementation of a method that is already defined in its superclass. The overridden method in the subclass should have the same signature as in the parent class.

    class Animal:
    def speak(self):
        return "Some sound"

class Dog(Animal):
    def speak(self):
        return "Woof"

class Cat(Animal):
    def speak(self):
        return "Meow"

dog = Dog()
print(dog.speak())  # Outputs: Woof

cat = Cat()
print(cat.speak())  # Outputs: Meow

    Note: if the signature are different, you can't interchangeably use objects of the subclasses (Dog and Cat) without considering their method signatures.

    Overloading:

    Overloading in many object-oriented programming languages typically refers to the ability to have multiple methods in the same scope with the same name but different signatures (i.e., different numbers or types of parameters). However, Python does not support this traditional form of method overloading. Instead, Python has its own unique approach to achieve similar functionality:

    1. Default Parameter Values:
      class MathOperation:
    def add(self, a, b, c=0):
        return a + b + c

operation = MathOperation()
print(operation.add(1, 2))    # Outputs 3 (using two arguments)
print(operation.add(1, 2, 3)) # Outputs 6 (using three arguments)
    1. Variable-Length Arguments:

      This refers to using *args and **kargs

    1. Operator Overloading:

      Python allows you to overload certain operators so that they have customized behavior with respect to your objects.

      class Point:
    def __init__(self, x=0, y=0):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Point(self.x + other.x, self.y + other.y)

p1 = Point(1, 2)
p2 = Point(3, 4)
result = p1 + p2
print(result.x, result.y)  # Outputs: 4 6

    Polymorphism:

    Polymorphism is a fundamental concept in object-oriented programming (OOP) that allows objects of different classes to be treated as objects of a common superclass. It is a key principle that enables an object to take on many forms.

  • Dunder (double underscoring) functions (magic methods or special methods)

    Dunders are typically inherited from the base class object. (The object class is the most base type in Python, and all classes implicitly inherit from it unless another class is explicitly specified). They are special predefined methods that you can define in your own classes to enrich them with certain behaviors.

    __init__:

    This method is the constructor for a class, initializing new instances.

    class MyClass:
    def __init__(self, value):
        self.value = value

# Using __init__
obj = MyClass(10)
obj.value  # Outputs: 10

    __call__:

    Allows an instance to be called like a function.

    class MyClass:
    def __call__(self, arg):
        return f"Called with {arg}"

# Using __call__
obj = MyClass()
obj('argument')  # Outputs: Called with argument

    __getitem__:

    Allows an object's instance to use the square brackets ([]) operator, enabling you to access elements as if the instance was a collection.

    class IndexableContainer:
    def __init__(self, data):
        self.data = data

    def __getitem__(self, key):
        return self.data[key]

# Using __getitem__ with a list
container = IndexableContainer(['zero', 'one', 'two', 'three'])
print(container[0])  # Outputs: zero
print(container[1])  # Outputs: one
print(container[2:4])  # Outputs: ['two', 'three']

# Using __getitem__ with a dictionary
container = IndexableContainer({'a': 'alpha', 'b': 'beta', 'c': 'gamma'})
print(container['a'])  # Outputs: alpha
print(container['b'])  # Outputs: beta
  • Variables and Methods

    Instance Variables:

    They are specific to each instance of a class; they represent the state or properties of a particular object, meaning each object can have different values for these variables.

    Class Variables:

    Class variables, on the other hand, are shared across all instances of a class. They are defined within the class but outside of any methods (including __init__) and are not specific to any instance. Changing the value of a class variable affects all instances at the same time.

    Regular Methods:

    Automatically take the instance as the first argument (self).

    Class Methods @classmethod:

    Automatically take the class as the first argument (cls). Use cases (Types):

    1. Change a class variable across all instances (a class method is bound to the class rather than its instances, so it modifies class variables that are shared across all instances).
    1. We can use them as alternative constructors (usually start with ‚Äúfrom‚Äù).

    Static Methods @staticmethod:

    Static methods behave like plain/regular functions, except they are grouped within a class. The primary reason to use static methods is organizational: they are logically associated with the class they are defined in, but they do not interact with the class or instance properties.

    class Employee:

    num_of_emps = 0  # CLASS VARIABLE
    raise_amt = 1.04  # CLASS VARIABLE

    def __init__(self, first, last, pay):  # CONSTRUCTOR
        self.first = first  # INSTANCE VARIABLE
